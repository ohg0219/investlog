# Todo Scanner

코드베이스 전체(또는 지정한 경로)를 스캔하여 TODO, FIXME, HACK, XXX, NOTE 등의 주석을 수집하고,
우선순위와 태그 기준으로 분류한 뒤 구조화된 목록으로 출력하는 도구입니다.

## Process

### 1. Arguments 파싱

`$ARGUMENTS`를 분석하여 다음 옵션을 추출한다:

- **path**: 검색할 경로 (생략 시 현재 디렉토리 전체 대상)
  - 예: `src/`, `lib/utils.ts`, `.`
- **--tag TAG**: 특정 태그만 필터링 (생략 시 전체 태그 검색)
  - 예: `--tag FIXME`, `--tag TODO`
- **--ext EXT**: 특정 확장자만 검색 (생략 시 주요 코드 파일 대상)
  - 예: `--ext ts`, `--ext py`

### 2. 태그 패턴으로 Grep 검색

아래 패턴으로 대상 경로를 검색한다. `--tag` 옵션이 지정된 경우 해당 태그만 검색한다:

- `FIXME` - 반드시 수정해야 하는 버그나 문제
- `HACK` - 임시방편 코드, 리팩터링 필요
- `TODO` - 구현 예정 기능이나 작업
- `XXX` - 주의가 필요한 위험한 코드
- `NOTE` - 중요 정보나 주의사항 메모

Grep 검색 시 파일명, 줄 번호, 매칭 내용을 함께 수집한다.

### 3. 결과 수집 및 분류

각 매칭 항목에서 다음 정보를 추출한다:

- 파일 경로
- 줄 번호
- 태그 종류 (FIXME / HACK / TODO / XXX / NOTE)
- 주석 내용 (태그 이후 텍스트)
- 작성자 정보 (있는 경우: `TODO(작성자): ...` 패턴)

우선순위 순서에 따라 정렬한다:

```
FIXME (1순위) > HACK (2순위) > TODO (3순위) > XXX (4순위) > NOTE (5순위)
```

### 4. 파일별 / 태그별 그룹핑

수집한 항목을 두 가지 관점으로 그룹핑한다:

- **태그별 그룹**: 우선순위가 높은 태그부터 항목 나열
- **파일별 그룹**: 동일 파일 내 항목을 줄 번호 순으로 나열

### 5. 정리된 결과 출력

아래 Output 형식에 따라 최종 결과를 출력한다.

## Output

### 요약 헤더

```
Todo Scanner 결과
검색 경로: {path}
태그 필터: {tag 또는 "전체"}
총 {N}건 발견  |  FIXME {n}  HACK {n}  TODO {n}  XXX {n}  NOTE {n}
```

### 태그별 섹션 (우선순위 순)

각 태그 섹션을 다음 형식으로 출력한다:

```
## FIXME ({n}건)
- [{파일경로}:{줄번호}] {주석 내용}
- [{파일경로}:{줄번호}] {주석 내용}

## HACK ({n}건)
- [{파일경로}:{줄번호}] {주석 내용}

## TODO ({n}건)
- [{파일경로}:{줄번호}] {주석 내용}
  작성자: {작성자} (있는 경우)

## XXX ({n}건)
- [{파일경로}:{줄번호}] {주석 내용}

## NOTE ({n}건)
- [{파일경로}:{줄번호}] {주석 내용}
```

### 파일별 요약

```
## 파일별 분포
{파일경로}: FIXME {n}, TODO {n}, ... (총 {n}건)
{파일경로}: HACK {n}, NOTE {n}, ... (총 {n}건)
```

## Error Cases

- **검색 경로가 존재하지 않는 경우**: `오류: '{path}' 경로를 찾을 수 없습니다.` 출력 후 종료
- **결과가 0건인 경우**: `'{path}'에서 해당 태그를 발견하지 못했습니다.` 출력
- **파일이 너무 많아 검색이 느린 경우**: `node_modules`, `.git`, `dist`, `build` 등 주요 제외 디렉토리는 자동으로 스킵한다
